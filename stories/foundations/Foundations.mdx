# Foundations

Design foundations that establish the visual language and core elements of the Loupe design system. These components define colors, typography, spacing, and brand elements.

## What are Foundation Components?

Foundation components establish the fundamental visual elements that all other components build upon. They provide:

- **Visual consistency** - Unified color palette and typography
- **Brand identity** - Logo and brand elements
- **Design tokens** - Standardized spacing, colors, and typography
- **Accessibility** - Color contrast and readable typography

## Component Categories

### Color System

- **Colors** - Color palette with semantic meanings and usage guidelines
- **Color tokens** - Design tokens for consistent color application

### Typography

- **Typography** - Font families, sizes, weights, and line heights
- **Text styles** - Predefined text styles for different contexts

### Spacing

- **Spacing** - Standardized spacing scale and usage patterns
- **Layout tokens** - Margins, padding, and gap values

### Shadows

- **Shadows** - Box shadow utilities for depth and elevation
- **Shadow tokens** - Standardized shadow values and usage patterns

### Brand Elements

- **Logo** - Brand logo and variations
- **Brand assets** - Official brand elements and guidelines

## Usage Guidelines

### When to use Foundation Components

- As the base for all design decisions
- When establishing brand consistency
- For creating new components
- When implementing design system updates

### Best Practices

- Always use foundation tokens instead of hardcoded values
- Follow the established color meanings and usage
- Use typography scales consistently
- Test color combinations for accessibility
- Maintain brand consistency across all applications

## Color System

### Primary Colors

Primary brand colors establish the core visual identity with blue, indigo, purple, and pink variations.

### Semantic Colors

Semantic colors provide meaning through success (green), warning (yellow), error (red), and info (blue) color coding.

### Neutral Colors

Neutral grayscale colors provide the foundation for text, backgrounds, and subtle UI elements.

## Typography Scale

### Font Families

Sans-serif fonts for body text and monospace fonts for code and technical content.

### Font Sizes

Comprehensive size scale from extra small (12px) to extra large (60px) for different content hierarchies.

### Font Weights

Weight variations from light (300) to extra bold (800) for emphasis and hierarchy.

## Spacing System

### Spacing Scale

Standardized spacing scale from 0 to 96 (0px to 384px) for consistent layout and component spacing.

## Shadow System

### Shadow Scale

Comprehensive shadow utilities from subtle (shadow-sm) to dramatic (shadow-2xl) for creating depth and visual hierarchy.

### Shadow Usage

- **Cards & Panels** - Use shadow-sm to shadow-lg for different elevation levels
- **Interactive Elements** - Apply hover and focus shadow states for better UX
- **Overlays & Modals** - Use shadow-xl and shadow-2xl for prominent elements

## Logo Usage

### Logo Variations

Primary, monochrome, and icon-only logo variations for different contexts and backgrounds.

### Logo Guidelines

Proper spacing, contrast, and usage guidelines ensure consistent brand representation.

## Accessibility

### Color Contrast

All color combinations meet WCAG AA standards with proper contrast ratios for text and UI elements.

### Typography Accessibility

Minimum font sizes, sufficient line heights, and clear hierarchy ensure readable typography.

## Design Tokens

### CSS Custom Properties

CSS custom properties provide consistent design tokens for colors, typography, and spacing across all components.
