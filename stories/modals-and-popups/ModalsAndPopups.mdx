# Modals & Popups

Modal dialogs, popovers, tooltips, and overlay components that provide contextual information and user interactions. These components help create layered interfaces and focused user experiences.

## What are Modal & Popup Components?

Modal and popup components provide overlay interfaces that appear above the main content. They offer:

- **Focused interactions** - Draw attention to specific actions or information
- **Contextual content** - Show relevant information without navigation
- **User guidance** - Provide help, instructions, and feedback
- **Layered interfaces** - Create depth and hierarchy in user interfaces

## Component Categories

### Modal Dialogs

- **BottomSheetDialog** - Full-screen bottom-anchored dialog for forms and multi-step workflows
- **Dialog** - Full modal dialogs for important actions
- **FormModal** - Modal for forms with header, scrollable content, and footer actions
- **Sheet** - Slide-out panel for forms, navigation, and secondary content
- **AlertDialog** - Confirmation and alert dialogs
- **Command** - Command palette and search interfaces

### Overlay Components

- **Popover** - Contextual floating content
- **Tooltip** - Hover-triggered information
- **HoverCard** - Rich hover content displays
- **DropdownMenu** - Action menus and selections

### Notification Components

- **Toast** - Temporary status messages
- **InfoHover** - Hover-triggered information displays

## Usage Guidelines

### When to use Modal & Popup Components

- For important actions that require user confirmation
- When providing contextual help and information
- For displaying additional content without navigation
- When creating command interfaces and search
- For showing notifications and status updates

### Best Practices

- Use modals sparingly for critical actions only
- Provide clear escape routes (close buttons, ESC key)
- Ensure proper focus management
- Test with keyboard navigation
- Consider mobile and touch interactions
