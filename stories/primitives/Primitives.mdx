# Primitives

The foundational building blocks of the Loupe design system. These are the essential UI components that form the backbone of all user interfaces.

## What are Primitives?

Primitives are the most basic, reusable UI components in our design system. They provide the essential building blocks that other components and pages are built from. These components are:

- **Accessible by default** - Built with accessibility best practices
- **Highly reusable** - Can be used across different contexts and applications
- **Consistent** - Follow the same design patterns and behavior
- **Composable** - Can be combined to create more complex components

## Component Categories

### Interactive Elements

- **Toggle** - Binary state controls
- **ToggleGroup** - Multiple selection controls
- **Command** - Command palette and search interfaces

### Navigation & Structure

- **Breadcrumb** - Navigation hierarchy indicators
- **Pagination** - Page navigation controls
- **Tabs** - Content organization and switching
- **Separator** - Visual content dividers

### Feedback & Communication

- **Alert** - Important messages and notifications
- **AlertDialog** - Modal confirmation dialogs
- **Dialog** - Modal content containers
- **Toast** - Temporary status messages
- **Tooltip** - Contextual help and information
- **HoverCard** - Rich hover content displays

### Data Display

- **Avatar** - User profile images and initials
- **Badge** - Status indicators and labels
- **IconBadge** - Icon-based status indicators
- **Progress** - Task completion indicators

### Layout & Containers

- **Card** - Content grouping containers
- **Collapsible** - Expandable content sections
- **Accordion** - Collapsible content lists
- **ScrollArea** - Custom scrollable containers

### Overlays & Positioning

- **Popover** - Contextual floating content
- **DropdownMenu** - Action menus and selections
- **Calendar** - Date selection interfaces

### Brand & Identity

- **Logo** - Brand identity components

## Usage Guidelines

### When to use Primitives

- As the foundation for custom components
- For standard UI patterns that need consistency
- When building forms, navigation, or data displays
- For any interface that requires accessibility compliance

### Best Practices

- Always use the most appropriate primitive for your use case
- Combine primitives to create complex interactions
- Follow the established patterns for each component
- Test accessibility with screen readers and keyboard navigation
- Use semantic HTML elements when possible
