# Buttons

Button components for user actions and interactions. These components provide consistent styling and behavior for triggering actions throughout the application.

## What are Button Components?

Button components are interactive elements that allow users to trigger actions. They provide:

- **Consistent styling** - Standardized appearance across the application
- **Multiple variants** - Different styles for different use cases
- **Accessibility** - Keyboard navigation and screen reader support
- **Visual feedback** - Hover, active, and disabled states

## Component Categories

### Standard Buttons

- **Button** - Primary action triggers with multiple variants and sizes
- **SplitButton** - Button with dropdown menu for primary and secondary actions
- **HoldButton** - Button requiring user to hold for confirmation before action

## Usage Guidelines

### When to use Button Components

- For primary and secondary actions
- When you need consistent action triggers
- For destructive actions that need confirmation
- When grouping related actions together

### Best Practices

- Use the appropriate variant for the action importance
- Provide clear, action-oriented labels
- Use destructive variants for dangerous actions
- Consider using HoldButton for critical actions that need confirmation
- Group related buttons using ButtonGroupWrapper

## Button Variants

### Default

Primary actions and important operations.

### Secondary

Secondary actions and less prominent operations.

### Destructive

Dangerous actions like delete or remove.

### Outline

Actions that need less visual weight.

### Ghost

Subtle actions that don't need strong emphasis.

### Link

Text-style buttons for less prominent actions.

## Button Sizes

- **2xs** - Extra extra small (16px)
- **xs** - Extra small (20px)
- **sm** - Small (24px)
- **default** - Default size (32px)
- **lg** - Large (40px)
- **icon** - Square icon-only buttons
