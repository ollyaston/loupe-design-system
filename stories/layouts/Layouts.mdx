# Layouts

Layout components provide the structural foundation for pages and applications. These components handle spacing, positioning, and responsive behavior.

## What are Layout Components?

Layout components define the structure and organization of user interfaces. They provide:

- **Consistent spacing** - Standardized margins, padding, and gaps
- **Responsive design** - Adaptive layouts for different screen sizes
- **Content organization** - Logical grouping and hierarchy
- **Navigation structure** - Sidebar, header, and content areas

## Component Categories

### Page Structure

- **Page** - Main page container with standard spacing
- **PageWrapper** - Enhanced page wrapper with additional features
- **FullLayout** - Full-screen layout without navigation
- **SidebarLayout** - Layout with collapsible sidebar navigation
- **TakeoverLayout** - Full-screen modal-style layouts

### Specialized Layouts

- **AccountLayout** - User account and profile page structure
- **ChatLayout** - Chat interface with message areas and input
- **DetailsSidebar** - Sidebar component for displaying property details with optional actions

## Usage Guidelines

### When to use Layout Components

- As the foundation for all pages and views
- When building consistent navigation structures
- For responsive design implementation
- When organizing content into logical sections

### Best Practices

- Choose the appropriate layout for your content type
- Maintain consistent spacing using the design system tokens
- Ensure layouts work across all device sizes
- Use semantic HTML structure
- Test with different content lengths

## Layout Patterns

### Standard Page Layout

Main page containers with headers, content areas, and proper spacing for consistent page structure.

### Sidebar Navigation Layout

Layouts with collapsible sidebar navigation provide organized navigation structures for complex applications.

## Responsive Behavior

All layout components are built with responsive design in mind:

- **Mobile-first approach** - Optimized for mobile devices
- **Breakpoint-aware** - Adapts to different screen sizes
- **Touch-friendly** - Appropriate sizing for touch interfaces
- **Performance optimized** - Efficient rendering and updates

## Accessibility

Layout components ensure proper accessibility:

- **Semantic structure** - Uses appropriate HTML elements
- **Keyboard navigation** - Logical tab order and focus management
- **Screen reader support** - Proper heading hierarchy and landmarks
- **Color contrast** - Meets WCAG guidelines

## Spacing System

Layouts use the design system's spacing tokens:

- **xs** - 4px (0.25rem)
- **sm** - 8px (0.5rem)
- **md** - 16px (1rem)
- **lg** - 24px (1.5rem)
- **xl** - 32px (2rem)
- **2xl** - 48px (3rem)
