# Forms

Form components and controls for data collection and user input. These components handle user interactions, validation, and data submission.

## What are Form Components?

Form components are specialized UI elements designed for data collection and user input. They provide:

- **Input validation** - Built-in validation patterns and error handling
- **User feedback** - Clear success, error, and loading states
- **Accessibility** - Proper labeling and keyboard navigation
- **Consistency** - Unified styling and behavior across all form elements

## Component Categories

### Text Input

- **Input** - Single-line text input fields
- **CurrencyInput** - Currency input for entering monetary values with symbol and decimal formatting
- **Textarea** - Multi-line text input areas
- **InputOTP** - One-time password input fields

### Selection Controls

- **Checkbox** - Binary selection controls
- **RadioGroup** - Single selection from multiple options
- **RadioBlocks** - Card-style radio button groups
- **Select** - Dropdown selection menus
- **Switch** - Toggle switches for binary states

### Range Controls

- **Slider** - Numeric range selection

### Form Structure

- **Form** - Form container with validation handling
- **FormActions** - Standardized form action buttons
- **Label** - Accessible form field labels

## Usage Guidelines

### When to use Form Components

- For any data collection interface
- When building user registration or profile forms
- For search and filter interfaces
- When implementing settings or configuration panels

### Best Practices

- Always provide clear labels for form fields
- Use appropriate input types for data validation
- Provide helpful error messages and validation feedback
- Group related fields logically
- Use consistent spacing and alignment

## Form Validation

Our form components support multiple validation patterns with built-in validation handling, error states, and user feedback.

## Accessibility Features

- **Proper labeling** - All inputs have associated labels
- **Keyboard navigation** - Full keyboard support for all controls
- **Screen reader support** - ARIA attributes for assistive technologies
- **Error announcements** - Validation errors are announced to screen readers
- **Focus management** - Logical tab order and focus indicators

## Common Patterns

### Registration Form

Forms support multi-column layouts with proper spacing and validation for user registration flows.

### Settings Panel

Forms can be organized into logical sections with grouped controls for settings and configuration panels.
